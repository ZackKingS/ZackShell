# ZackShell 需求文档(PRD)

| 项 | 内容 |
|---|---|
| 项目名称 | ZackShell |
| 文档版本 | v1.0 |
| 编写日期 | 2026-06-13 |
| 目标平台 | Windows 10 / 11(x64),架构允许后续跨平台 |
| 定位 | 一款集成 SSH 终端、SFTP 文件管理与服务器实时监控的桌面客户端,对标 FinalShell |

---

## 1. 项目背景与目标

### 1.1 背景
运维 / 开发人员日常需要:登录远程 Linux 服务器执行命令、上传下载文件、查看服务器实时负载。现有工具各有侧重(PuTTY 只有终端、WinSCP 只有文件、xshell 商业收费),FinalShell 把三者整合并加入实时监控面板,体验好但为 Java 实现、体积大、部分功能收费。

### 1.2 目标
做一款 **Windows 原生体验** 的开源替代:
- 一个界面内完成「连接 → 终端操作 → 文件传输 → 服务器监控」全流程。
- 监控数据通过 SSH 远程采集,**目标服务器无需安装任何 agent**。
- 启动快、不卡顿、中文友好。

### 1.3 非目标(本期不做)
- 不做 Web 版 / 移动端。
- 不做 Telnet、串口、RDP、VNC(可作为远期扩展)。
- 不做团队协作 / 多人共享会话。
- 不做服务器端管理(如 Docker、数据库可视化管理)。

---

## 2. 用户与场景

### 2.1 目标用户
- 后端开发、运维工程师、个人服务器 / VPS 用户。
- 习惯命令行,但希望文件管理和监控可视化。

### 2.2 典型场景
1. **日常登录**:打开软件 → 选已保存会话 → 一键登录 → 在终端执行命令。
2. **文件传输**:把本地构建产物拖到服务器某目录;把服务器日志下载到本地查看。
3. **排障监控**:服务器变慢,打开监控面板看 CPU / 内存 / 网络 / 进程占用,快速定位。
4. **多机管理**:同时连多台服务器,标签页切换。
5. **跳板登录**:通过堡垒机跳转到内网服务器。

## 3. 功能需求

> 优先级:**P0 = 必须有(MVP)**,**P1 = 重要**,**P2 = 增强**。

### 3.1 连接与会话管理

| 编号 | 功能 | 优先级 | 说明 |
|---|---|---|---|
| C-1 | SSH 密码登录 | P0 | 主机、端口、用户名、密码 |
| C-2 | SSH 密钥登录 | P0 | 支持 RSA/ED25519,带 passphrase |
| C-3 | 会话保存与管理 | P0 | 增/删/改/查,持久化到本地 |
| C-4 | 会话分组 | P1 | 按文件夹分组组织会话 |
| C-5 | 凭据加密存储 | P0 | 密码/密钥不得明文落盘 |
| C-6 | 快速连接 | P1 | 不保存,临时输入信息直接连 |
| C-7 | 跳板机 / ProxyJump | P2 | 通过中间主机跳转 |
| C-8 | 连接保活 | P1 | keepalive 心跳,断线提示 |

### 3.2 终端

| 编号 | 功能 | 优先级 | 说明 |
|---|---|---|---|
| T-1 | 交互式终端 | P0 | 完整 xterm/VT100 兼容 |
| T-2 | UTF-8 / 中文 | P0 | 不乱码 |
| T-3 | 复制 / 粘贴 | P0 | 选中复制、右键/快捷键粘贴 |
| T-4 | 窗口自适应 resize | P0 | 终端尺寸随窗口变化同步到远端 |
| T-5 | 多标签会话 | P1 | 同时打开多个终端 |
| T-6 | 终端分屏 | P2 | 水平 / 垂直拆分 |
| T-7 | 主题 / 字体 / 字号 | P1 | 配色方案、字体可调 |
| T-8 | 搜索终端内容 | P2 | Ctrl+F 在输出中查找 |
| T-9 | 命令历史与补全 | P2 | 输入框历史记忆 |

### 3.3 文件管理(SFTP)

| 编号 | 功能 | 优先级 | 说明 |
|---|---|---|---|
| F-1 | 远程目录浏览 | P0 | 目录树 + 文件列表(名称/大小/类型/时间/权限/属主) |
| F-2 | 上传 / 下载 | P0 | 带进度,支持大文件 |
| F-3 | 拖拽传输 | P1 | 从资源管理器拖入上传 |
| F-4 | 新建 / 删除 / 重命名 | P0 | 文件与目录 |
| F-5 | 修改权限(chmod) | P1 | 可视化权限编辑 |
| F-6 | 在线编辑文本文件 | P1 | 双击打开 → 编辑 → 保存回传 |
| F-7 | 传输队列与断点续传 | P2 | 队列管理、失败重试 |
| F-8 | 本地 / 远程双栏对比 | P2 | 类 WinSCP 双栏 |

### 3.4 服务器监控(核心差异化)

| 编号 | 功能 | 优先级 | 说明 |
|---|---|---|---|
| M-1 | CPU 使用率 | P0 | 实时百分比,基于 /proc/stat 差值 |
| M-2 | 内存 / 交换分区 | P0 | 已用/总量、百分比 |
| M-3 | 磁盘挂载点使用 | P0 | 各分区 可用/总量 |
| M-4 | 网络上下行流量 | P0 | 实时曲线,基于 /proc/net/dev 差值 |
| M-5 | 进程列表 | P1 | 按 CPU/内存 排序 Top N |
| M-6 | 系统信息 | P1 | 内核、运行时长、负载、主机名 |
| M-7 | 网络延迟(ping) | P2 | 到目标主机的延迟曲线 |
| M-8 | 采集频率可配 | P1 | 默认 2s,可调 |

### 3.5 系统设置

| 编号 | 功能 | 优先级 | 说明 |
|---|---|---|---|
| S-1 | 主密码保护 | P1 | 启动时解锁凭据库 |
| S-2 | 外观设置 | P1 | 主题、字体、语言(中/英) |
| S-3 | 配置导入 / 导出 | P2 | 备份会话 |
| S-4 | 自动更新 | P2 | 检测新版本 |

## 4. 非功能需求

| 类别 | 要求 |
|---|---|
| 性能 | 终端输入延迟 < 50ms;监控面板刷新不阻塞终端;空闲内存占用尽量低 |
| 安全 | 凭据加密存储(系统凭据库或主密码 AES);主机密钥校验(known_hosts);渲染进程不直连网络 |
| 稳定性 | 单连接异常不影响其他连接;断线自动提示 / 可重连 |
| 兼容性 | 兼容主流 Linux 发行版(Debian/Ubuntu/CentOS);终端兼容常见 TUI 程序(vim/top/tmux) |
| 易用性 | 中文界面;常用操作 ≤ 3 步;快捷键支持 |
| 可维护性 | 模块化、UI 与协议层解耦,便于后续接入其他协议 |

## 5. 验收标准(MVP)

完成以下即认为 MVP 可用:
1. 能保存会话并一键 SSH 登录截图中的 Debian 服务器,终端可正常运行 `top`、`vim`,中文不乱码。
2. 能通过 SFTP 浏览目录、上传 / 下载文件并显示进度。
3. 监控面板显示的 CPU / 内存 / 磁盘 / 网络数据与服务器实测一致,且每 2 秒刷新。
4. 凭据加密存储,重启后无需重新输入即可连接。
5. 关闭一个连接 / 连接断开不会导致整个程序崩溃。

## 6. 里程碑

| 阶段 | 内容 | 交付 | 预估 |
|---|---|---|---|
| Phase 0 | 脚手架 + 本地终端 | 可输入的终端窗口 | 3–5 天 |
| Phase 1 | SSH 终端 MVP | 能登录远程跑命令 | 1–2 周 |
| Phase 2 | 会话管理 + 凭据加密 | 保存并一键连接 | 1 周 |
| Phase 3 | SFTP 文件管理 | 传输 + 在线编辑 | 1.5–2 周 |
| Phase 4 | 监控面板 | 全套实时监控 | 2–3 周 |
| Phase 5 | 进阶(多标签/转发/跳板/主题/同步) | 按需 | 弹性 |
| Phase 6 | 打包发布 | Windows 安装包 | 3–5 天 |

> 单人开发,Phase 0–4 完成即为可用产品,约 8–10 周。

## 7. 风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| 终端兼容性(TUI 程序显示异常) | 高 | 用成熟的 xterm.js,严格处理 PTY 尺寸与转义序列 |
| 监控采集命令在不同发行版差异 | 中 | 优先读 /proc(通用),df/free 做兜底,做发行版探测 |
| 凭据安全 | 高 | 系统凭据库 + 主密码,绝不明文 |
| 大文件传输性能 | 中 | 流式传输 + 进度事件,后续做断点续传 |
| Electron 体积大 | 低 | 接受;远期可评估迁移 Tauri |
